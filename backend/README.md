# Iris Insight Dashboard - Backend

The backend serves two purposes. First, it exposes dashboard ready JSON artifacts generated by the notebook. Second, it loads the saved SVM model and provides a prediction endpoint for new Iris measurements. The SVM model was selected based on cross validation accuracy, while Random Forest feature importance is served separately as an interpretability artifact.

## Running locally

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --reload-dir app
```

The `--reload-dir app` flag scopes the file watcher to your source code only, so it does not thrash on the entire `.venv` directory.

The API will be available at `http://127.0.0.1:8000`.
Interactive docs are at `http://127.0.0.1:8000/docs`.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/` | Confirm the API is running |
| GET | `/health` | Health check |
| GET | `/metrics` | Best model accuracy and classification metrics |
| GET | `/model-comparison` | Cross-validation accuracy across all trained models |
| GET | `/confusion-matrix` | Confusion matrix for the test set |
| GET | `/classification-report` | Per-class precision, recall, and F1 |
| GET | `/feature-importance` | Random Forest feature importance scores |
| GET | `/dataset-summary` | Iris dataset descriptive statistics |
| GET | `/pca-projection` | 2D PCA projection of all 150 samples |
| GET | `/sample-predictions` | Predictions on the held-out test set |
| POST | `/predict` | Predict species from new Iris measurements |

## Predict endpoint

Send a POST request to `/predict` with a JSON body:

```json
{
  "sepal_length": 6.7,
  "sepal_width": 3.1,
  "petal_length": 5.6,
  "petal_width": 2.4
}
```

All four values must be floats between 0.1 and 10.0.
